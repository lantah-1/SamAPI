# 开发计划

## 阶段 1：切换型路由 MVP

状态：已实现基础版本。

交付项：

- React + TypeScript + TailwindCSS 控制台。
- Node 本地 API。
- JSON 文件数据库，默认 `data/samapi.json`。
- `SAMAPI_DATA_DIR` 指定数据库目录。
- 站点管理，支持多地址；模型管理暂缓到后续阶段。
- 密钥生成、启停、删除。
- 上游 API Key 分组管理，支持按供应商保存多 Key，并通过 `/models` 发现可用模型。
- Header 模版管理。
- 切换型路由管理。
- `/proxy` 本地模型代理，通过请求体 `model` 匹配路由。
- 日志管理，默认记录模型代理请求，支持展开查看详细请求信息。

验收标准：

- 管理台可以新增站点、Header、密钥和切换型路由。
- 路由创建时供应商、地址、模型是级联选择。
- endpoint 默认 `message`，实际值为 `messages`。
- 下游客户端可以通过路由名访问本地代理。
- 密钥只在生成时显示明文，数据库只保存哈希。
- 日志列表折叠时可看到模型、供应商、UA、成功/失败状态，展开时可看到请求头、请求体、上游地址、响应预览和错误信息。

## 阶段 2：分组型路由

交付项：

- 模型组管理：同一模型可绑定多个供应商和地址。
- 分组路由管理：选择模型组、endpoint、Header 规则和策略。
- 策略执行器：稳定优先、轮流调用、随机调用。
- 失败转移：失败后自动尝试组内其他候选。
- 运行状态：round-robin 游标、供应商失败计数、短时间熔断。

验收标准：

- 单个分组路由可关联多个供应商。
- 任意供应商失败时自动尝试下一个可用候选。
- 全部候选失败时返回清晰错误。
- 策略行为可通过日志或测试验证。

## 阶段 3：可靠性与观测

交付项：

- 请求日志和调用统计。
- 上游健康检查。
- 失败原因、耗时、状态码聚合。
- API Key 调用维度统计。
- 路由级限流和并发控制。

验收标准：

- 管理台可查看每个路由的成功率和最近错误。
- 不可用供应商会被短时间跳过。
- 可以按 key 或 route 定位异常调用。

## 阶段 4：数据库升级

交付项：

- SQLite 持久化实现。
- JSON 到 SQLite 的迁移脚本。
- 数据库备份和导入导出。

验收标准：

- 旧版 `samapi.json` 可迁移。
- 重启服务后数据完整。
- 支持并发读写场景下的数据一致性。

## 阶段 5：部署与安全

交付项：

- 单进程生产启动脚本。
- 管理台登录保护。
- CORS 白名单。
- 敏感环境变量检查。
- Dockerfile 或二进制打包方案。

验收标准：

- 可在本机或内网服务器稳定运行。
- 管理 API 不暴露给未授权访问。
- 密钥和 Header 不在日志中泄漏。
