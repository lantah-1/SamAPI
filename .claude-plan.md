## 依赖注入策略（关键）
- `store` 是 `index.ts` 里的单例 `new JsonStore()`。为避免每个模块各自 `import { store }` 造成循环依赖，采用**工厂函数**模式：
  - 有 store 依赖的模块导出 `createXxx(store)` 返回一组闭包函数（与现有 `createAuth(store)` 完全一致的模式）。
  - 无 store 依赖的纯函数模块（util/convert/providers/proxy-path）直接 `export function`。
- `index.ts` 里组装：`const accountCheck = createAccountCheck(store); const routing = createRouting(store); const discovery = createModelDiscovery(store); const apiHandler = createApiHandler({store, auth, accountCheck, discovery}); const proxyHandler = createProxyHandler({store, routing, ...})`。

## 风险与缓解
- **循环依赖**：util 只被上层引用、不反向引用；providers 不引用 account-check/routing；handlers 在最顶层。方向单一。
- **`this`/闭包语义**：工厂模式与现有 `createAuth` 一致，不涉及 class 实例，安全。
- **每步都 typecheck**：任何一步断了立即定位，不会累积。
- 全程不改逻辑，diff 应为纯移动（可用 git diff 按函数体核对）。

## store.ts 拆分（2004 行）
store.ts = 前 ~772 行**模块级纯函数**（临时账号解析/归一化、模型列表解析、配额解析、Grok/Codex 账号识别、settings 归一化等）+ 后 ~1230 行 `class JsonStore`（52 个方法，120 处 `this.db`/`this.persist`）。

- **class 本身不拆**：52 个方法共享 `this.db` + `this.persist()`，强行拆成多文件需要 mixin 或大改，违背"零行为变更"。保持 class 内聚。
- **只抽出模块级纯函数**到 `store/` 子目录，class 从这些模块 import：
```
server/store/
  index.ts          # class JsonStore（保留），re-export 供 `./store.js` 现有 import 路径不变
  parse-account.ts  # temporaryAccountLineFields,secretFromLine,cookieFromLine,grokAuthRecord*,
                    #   accountTypeFrom*,principalType*,firstStringField,stringField 等账号解析
  parse-quota.ts    # numberLike,quotaStageFromRecord,accountHasUsableQuota,collectQuotaStages,
                    #   availabilityFrom*,percentLike 等配额解析
  parse-model.ts    # normalizeModelList,modelMatchesRule,matchRuleTokens,modelMatchTokens,modelsFromRecord
  normalize.ts      # normalizeSettings,normalizeGroupStrategy,normalizeProxyConfig,normalizeBaseUrl,
                    #   hashSecret,now,常量 DEFAULT_SETTINGS/THEME_IDS/ENDPOINTS/*_BASE_URL 等
  jwt.ts            # jwtPayload,accountIdFromIdToken,emailFromIdToken（store 侧副本，与 providers 侧独立）
```

- **import 路径保持不变**：现有代码 `import { JsonStore, parseHeaderTemplate } from "./store.js"`。做法：把 `server/store.ts` 改为 `server/store/index.ts`，其余不变；或保留 `server/store.ts` 作为 re-export barrel。选后者更稳（`store.ts` → `export * from "./store/index.js"` 之类），避免动 index.ts 的 import。
  - 待定：`parseHeaderTemplate` 也从 store 导出，需确认它归到哪个子模块并继续从 barrel re-export。
